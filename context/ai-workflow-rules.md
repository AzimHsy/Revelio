# AI Workflow Rules

## Approach

Build Revelio incrementally using a spec-driven workflow. These context files define what to
build, how to build it, and the current state of progress. Always implement against these
specs — do not infer or invent behavior from scratch. When something is missing, resolve it
in the specs first.

## Scoping Rules

- Work on one feature unit at a time (e.g. "element selection", "GSAP extractor",
  "Claude call", "result rendering").
- Prefer small, verifiable increments over large speculative changes.
- Do not combine work across world boundaries (injected / content / background / sidepanel)
  in a single implementation step.

## When to Split Work

Split an implementation step if it combines:

- UI changes and runtime-extraction changes
- More than one world boundary (e.g. content script + background + side panel at once)
- Behavior not clearly defined in the context files

If a change cannot be verified end to end quickly (load the extension, inspect a real site,
see a result), the scope is too broad — split it.

## Handling Missing Requirements

- Do not invent product behavior not defined in the context files.
- If a requirement is ambiguous, resolve it in the relevant context file before
  implementing.
- If a requirement is missing, add it as an open question in `progress-tracker.md` before
  continuing.

## Protected Files

Do not modify the following unless explicitly instructed:

- The MV3 manifest config generated by the Vite CRX plugin — edit the source config, not the
  generated output.
- Any third-party library internals.

## Keeping Docs in Sync

Update the relevant context file whenever implementation changes:

- System boundaries or the world model → `architecture.md`
- Storage model decisions → `architecture.md`
- Code conventions or standards → `code-standards.md`
- Feature scope → `project-overview.md`

## Before Moving to the Next Unit

1. The current unit works end to end within its defined scope (real site → result in the
   panel).
2. No invariant defined in `architecture.md` was violated — especially the world-boundary
   and API-key rules.
3. `progress-tracker.md` reflects the completed work.
4. `npm run build` passes.
